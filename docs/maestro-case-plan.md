# Logithon CaseOps Maestro Case Plan

This project is designed for Track 1: UiPath Maestro Case. The case is not a fixed BPMN-only process because cargo loading exceptions are dynamic: missing evidence, imbalance, hazmat freight, damaged cargo, low confidence, and human override decisions can change the next step.

## UiPath Role

UiPath Maestro Case is the orchestration and governance layer:

- Case plan, stages, stage rules, re-entry, SLAs, and personas are modeled in Studio Web.
- Case App gives case workers and supervisors a live view of assigned work, status, SLA badges, and audit history.
- Case Instance Management gives operators pause, resume, cancel, migrate, retry, and incident control.
- Action apps or human action tasks capture supervisor decisions.
- API Workflows call the external cargo-risk service in this repo.
- Data Fabric or VDO can trigger cases from warehouse/TMS records.

The local React dashboard is a demo surface and custom coded case app prototype. The actual competition implementation should show the same lifecycle running through UiPath Automation Cloud.

## Case Entity

Entity name: `CargoLoadSafetyCase`

Case key: external key mapped from `shipment_id`. Use `LCOPS` as the system-prefix fallback when Maestro is the source of record.

Input fields are hydrated from Data Fabric, VDO, an API trigger, or the live intake panel:

- `shipment_id`
- `warehouse_id`
- `dock_id`
- `truck_type`
- `truck_capacity_kg`
- `truck_dimensions`
- `cargo_items`
- `weight_distribution`
- `evidence`
- `metadata`

## Stages

| Stage | Type | Required | Rule intent |
| --- | --- | --- | --- |
| Shipment Intake | Primary | Yes | Starts when a case is created and validates the cargo request. |
| Cargo Vision Review | Primary | Yes | Checks dock evidence and cargo metadata. |
| 3D Load Plan Generation | Primary | Yes | Creates a deterministic box placement plan from truck size and cargo dimensions. |
| Load Risk Analysis | Primary | Yes | Runs the coded risk service and writes risk outputs. |
| Human Supervisor Approval | Primary | Conditional | Activates when risk or uncertainty requires approval. |
| Dispatch Instruction | Primary | Yes | Creates loading controls and final dispatch outcome. |
| Evidence Rework | Secondary | No | Activates when missing evidence is detected and returns to origin. |
| Dispatch Blocked | Secondary | No | Interrupting exception path for rejected or blocked dispatch. |
| Closed | Primary | Yes | Terminal completed state. |

## Task I/O Contracts

The API exposes the full machine-readable contract at:

```text
GET /api/maestro/case-plan
GET /api/maestro/entity-schema
GET /api/maestro/task-contracts
```

The JSON artifact for import/reference is:

```text
uipath/maestro-case-plan.json
```

## Demo Flow

1. Create or select a cargo loading case.
2. Maestro enters `Shipment Intake`.
3. API Workflow calls `POST /api/analyze-cargo`.
4. API Workflow calls `POST /api/load-plan`.
5. API Workflow calls `POST /api/risk-score`.
6. If `requires_human_review` is true, Maestro enters `Human Supervisor Approval`.
7. Human action task writes the supervisor decision through `POST /api/human-decision`.
8. API Workflow calls `POST /api/dispatch-instructions`.
9. Maestro closes the case or routes to a secondary exception stage.

## What Is Real vs. Simulated

Real in this repo:

- Working Express API.
- Working React dashboard.
- Live case creation from user-entered cargo data.
- Deterministic cargo load-plan generation from truck dimensions and item dimensions.
- Deterministic cargo risk engine.
- Human approval endpoint.
- Dispatch instruction endpoint.
- Maestro-aligned case plan, entity schema, and task contracts.

Simulated until connected to UiPath Automation Cloud:

- Native Maestro Case runtime.
- Case App generated by Maestro.
- Action app tasks.
- Data Fabric or VDO trigger.
- UiPath authentication and tenant deployment.
- Real vision model or warehouse/TMS integrations.
